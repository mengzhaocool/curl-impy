## 一、项目概述

**本项目 ≠ lexiforest/curl-impersonate！** 独立定制版，在 lexiforest 基础上扩展了自定义浏览器注册、ECH GREASE、ALPS 新 codepoint、N-API 绑定等功能。

### 三大核心 API

| API | 功能 | 状态 |
|-----|------|------|
| `curl_easy_impersonate_list()` | 列出可模拟的浏览器（内置 + 自定义） | ✅ 已验证通过 |
| `curl_easy_impersonate_register(target, json_config)` | 动态注册自定义浏览器指纹 | ✅ 已验证通过 |
| `curl_easy_impersonate(curl, target, default_headers)` | 使用指定浏览器指纹发起请求 | ✅ 已验证通过 |

### 验证结果

- ✅ 38 个内置浏览器目标（chrome99~chrome146, edge99/101, firefox133~147, safari153~2601, okhttp4_android, tor145）
- ✅ 自定义浏览器注册+使用成功
- ✅ HTTPS 请求成功（curl_easy_perform → HTTP 200）
- ✅ DLL 只依赖 Windows 系统 DLL（WLDAP32, bcrypt, CRYPT32, WS2_32, IPHLPAPI, KERNEL32）

**关键修复记录：**
1. BUILD_CURL_EXE=OFF 跳过src/编译
2. nghttp2 需 `-DBUILD_STATIC_LIBS=ON`（否则 nghttp2_static alias 不存在）
3. nghttp3 用 `-DENABLE_SHARED_LIB=OFF -DENABLE_STATIC_LIB=ON`（变量名不同于nghttp2）
4. NGHTTP2_STATICLIB/NGTCP2_STATICLIB/NGHTTP3_STATICLIB 宏防止 __declspec(dllimport)
5. WHOLEARCHIVE 用 `$<LINK_LIBRARY:WHOLE_ARCHIVE,...>` CMake generator expression
6. **CURLOPT_HTTPBASEHEADER 缺失 case handler → setopt_slist() 中添加 case**
7. **impersonations[] 无 NULL 哨兵 → 用 num_impersonations 计数遍历**
8. **MSVC C89 兼容 → for(int i=...) 改为 int i; 声明**
9. **USE_ECH 未启用 → CURLOPT_ECH 调用加 #ifdef USE_ECH 保护**

### 2026-05-15 Step 7 完成
- **关键修复：CURLOPT_HTTPBASEHEADER (opt=11000) 在 setopt_slist() 中缺失 case handler**
  - 根因：CURLOPT_HTTPBASEHEADER 被 dispatch 到 setopt_slist() 函数，但该函数中没有对应 case
  - 修复：在 setopt.c 的 setopt_slist() 函数中添加 `case CURLOPT_HTTPBASEHEADER: data->state.base_headers = slist; break;`
  - 注意：base_headers 字段在 UrlState 结构体中（不是 UserDefined），需用 `data->state.base_headers`
- **DLL路径问题：运行时加载了旧版DLL**
  - 根因：项目根目录存在旧的 libcurl-impersonate.dll，Windows优先从exe所在目录加载
  - 修复：每次构建后同时复制到项目根目录和 output/ 目录
