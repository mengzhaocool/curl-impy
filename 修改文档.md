# curl-impersonate 修改文档

> 基于 lexiforest/curl-impersonate 补丁分析，记录修改的文件、函数、目的和意图。
> **不记录具体代码修改内容**，只记录修改的"什么"和"为什么"。
> 适用于 curl 8.20.0 源码直接修改。

---

## 一、BoringSSL 修改（boringssl.patch）

### 1.1 CMakeLists.txt
- **目的**：Windows构建适配 + 移除不需要的组件
- **修改点**：
  - clang-cl 添加 `-Wno-deprecated` 编译标志，避免Windows平台废弃API警告
  - 移除 `ssl/test` 子目录（不需要SSL测试）
  - 移除 `ssl_test` 可执行目标
  - 移除 `bssl` 命令行工具目标
  - 移除 `run_tests` 目标（Go测试）
  - 移除 `bssl` 的 install 目标

### 1.2 crypto/cipher/e_tls.cc
- **目的**：添加 AES-256-CBC + SHA256/SHA384 组合的 AEAD 实现
- **为什么**：TLS 1.2 密码套件（如 TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA384）需要这些组合，原版BoringSSL只支持SHA1
- **修改点**：
  - 添加 `aead_aes_256_cbc_sha256_tls` 和 `aead_aes_256_cbc_sha384_tls` 两个AEAD实现
  - 导出 `EVP_aead_aes_256_cbc_sha256_tls()` 和 `EVP_aead_aes_256_cbc_sha384_tls()`

### 1.3 crypto/fipsmodule/ec/p256_test.cc + third_party/fiat/p256_64.h
- **目的**：Windows MSVC兼容性修复
- **为什么**：p256_64.h 的内联汇编使用GNU风格（__GNUC__ + __x86_64__），MSVC编译时条件判断为true但汇编语法不兼容
- **修改点**：添加 `!defined(OPENSSL_WINDOWS)` 条件，Windows平台跳过内联汇编

### 1.4 crypto/rand/windows.cc
- **目的**：兼容旧版Windows（无bcryptprimitives.dll）
- **为什么**：ProcessPrng只在较新Windows可用，回退到RtlGenRandom(SystemFunction036)
- **修改点**：
  - 添加 `RtlGenRandomFunction` 类型和全局变量
  - `init_processprng()` 中 ProcessPrng 失败时回退到 advapi32 的 SystemFunction036
  - `CRYPTO_sysrand()` 中支持两种随机源

### 1.5 include/openssl/aead.h
- **目的**：导出新AEAD函数声明
- **修改点**：添加 `EVP_aead_aes_256_cbc_sha256_tls` 和 `EVP_aead_aes_256_cbc_sha384_tls`

### 1.6 include/openssl/nid.h
- **目的**：添加FFDHE密钥交换组的NID
- **为什么**：支持DHE密码套件的密钥交换
- **修改点**：添加 `NID_ffdhe2048`(966) 和 `NID_ffdhe3072`(967)

### 1.7 include/openssl/ssl.h
- **目的**：添加TLS指纹伪装所需的BoringSSL API
- **修改点**：
  - `SSL_CTX_set_extension_order()` — 设置TLS扩展的自定义顺序
  - `SSL_CTX_set_key_usage_check_enabled()` — 可选禁用密钥用途检查（某些浏览器不检查）
  - `SSL_GROUP_FFDHE2048`(0x0100) 和 `SSL_GROUP_FFDHE3072`(0x0101) — DHE组常量
  - `SSL_CTX_set_delegated_credentials()` — 设置委托凭证签名算法
  - `SSL_set_record_size_limit()` / `SSL_CTX_set_record_size_limit()` — 设置record_size_limit扩展值
  - `SSL_set_key_shares_limit()` / `SSL_CTX_set_key_shares_limit()` — 控制key share数量（1-3个）

### 1.8 include/openssl/target.h
- **目的**：添加LoongArch64架构支持
- **修改点**：添加 `__loongarch64` 条件定义 `OPENSSL_LOONGARCH64` 和 `OPENSSL_64_BIT`

### 1.9 include/openssl/tls1.h
- **目的**：添加TLS扩展类型常量
- **修改点**：
  - `TLSEXT_TYPE_record_size_limit`(28) — record_size_limit扩展
  - `TLSEXT_cert_compression_zstd`(3) — zstd证书压缩算法

### 1.10 ssl/extensions.cc
- **目的**：实现TLS指纹伪装的核心扩展控制
- **为什么**：不同浏览器的ClientHello中扩展顺序、key share数量、是否发送record_size_limit等各不相同，需要精确控制
- **修改点**：
  - `ssl_setup_key_shares()` — 支持自定义key_shares_limit(1-3个key share)，默认2个
  - `ssl_ext_key_share_parse_serverhello()` — 支持第三个key share匹配
  - `ext_delegated_credential_add_clienthello()` — 实现自定义delegated_credentials发送逻辑
  - 新增 `record_size_limit` 扩展处理函数（add_clienthello/parse_serverhello/parse_clienthello/add_serverhello）
  - 将record_size_limit添加到kExtensions数组
  - 新增 `ssl_set_extension_order()` — 解析自定义扩展顺序字符串，生成extension_permutation数组
  - `ssl_add_clienthello_tlsext_inner()` 和 `ssl_add_clienthello_tlsext()` — 跳过索引255的扩展（自定义顺序中不存在的扩展）

### 1.11 ssl/handshake_client.cc
- **目的**：TLS 1.3密码套件顺序控制 + DHE密钥交换 + 签名算法去重检查移除
- **为什么**：不同浏览器TLS 1.3密码套件顺序不同（Chrome/Firefox/Safari），需要精确匹配
- **修改点**：
  - `ssl_write_client_cipher_list()` — 添加Firefox/Safari/其他密码套件顺序常量，根据cipher_order配置选择
  - `do_start_connect()` — 添加 `ssl_set_extension_order(hs)` 调用
  - `do_read_server_key_exchange()` — 添加DHE密钥交换参数解析（dh_p/dh_g/dh_Ys）
  - `do_send_client_key_exchange()` — 添加DHE密钥交换实现（DH_generate_key + DH_compute_key）
  - `ssl_cert_check_key_usage()` — 可选禁用key_usage检查（key_usage_check_enabled）

### 1.12 ssl/internal.h
- **目的**：添加新常量和结构体字段
- **修改点**：
  - 重新分配算法位标志：`SSL_kDHE=0x02`, `SSL_kECDHE=0x04`, `SSL_kPSK=0x08`, `SSL_kGENERIC=0x10`
  - 添加 `SSL_SHA384=0x04`，调整 `SSL_AEAD=0x08`
  - `SSL_HANDSHAKE` — key_shares扩展到3个，添加dh_p/dh_g数组
  - `SSL_CONFIG` — 添加 extension_order/cipher_order/key_usage_check_enabled/delegated_credentials/record_size_limit/key_shares_limit
  - `ssl_ctx_st` — 同上对应字段

### 1.13 ssl/ssl_cipher.cc
- **目的**：添加DHE/CBC/SHA384等密码套件定义
- **为什么**：浏览器支持这些套件，指纹需要匹配
- **修改点**：
  - 添加DHE密码套件：DHE_RSA_WITH_AES_128/256_SHA, DHE_RSA_WITH_AES_128/256_SHA256, DHE_RSA_WITH_AES_128/256_GCM
  - 添加RSA CBC SHA256/SHA384套件
  - 添加ECDHE 3DES和ECDHE SHA384套件
  - 添加SHA256/SHA384 MAC别名
  - `ssl_cipher_get_evp_aead()` — 支持AES256+SHA256/SHA384组合
  - `ssl_create_cipher_list()` — 添加DHE和更多legacy套件到列表
  - `SSL_CIPHER_get_digest_nid()` / `SSL_CIPHER_description()` — 支持SSL_SHA384

### 1.14 ssl/ssl_key_share.cc
- **目的**：注册FFDHE密钥交换组
- **修改点**：kNamedGroups数组添加ffdhe2048和ffdhe3072

### 1.15 ssl/ssl_lib.cc
- **目的**：实现新增API + 配置从SSL_CTX复制到SSL
- **修改点**：
  - `SSL_new()` — 复制extension_order/cipher_order/key_usage_check_enabled/key_shares_limit/delegated_credentials/record_size_limit
  - `SSL_CTX_set_cipher_list()` — 保存cipher_order字符串
  - 新增 `SSL_set/SSL_CTX_set_record_size_limit()`
  - 新增 `SSL_set/SSL_CTX_set_key_shares_limit()`
  - 新增 `SSL_CTX_set_extension_order()` 和 `SSL_CTX_set_key_usage_check_enabled()`

### 1.16 ssl/ssl_privkey.cc
- **目的**：移除签名算法唯一性检查
- **为什么**：Safari 15发送重复的算法偏好，原版BoringSSL会拒绝
- **修改点**：
  - 删除 `compare_uint16_t` 和 `sigalgs_unique` 函数
  - `set_sigalg_prefs()` — 移除 sigalgs_unique 检查
  - 新增 `SSL_CTX_set_delegated_credentials()` 实现

## 二、ngtcp2 修改（ngtcp2.patch）

### 2.1 lib/includes/ngtcp2/ngtcp2.h
- **目的**：添加原始传输参数设置API
- **为什么**：QUIC传输参数需要精确控制字节级内容以匹配浏览器指纹
- **修改点**：
  - 添加 `ngtcp2_transport_params_raw` 结构体（base指针+len）
  - 声明 `ngtcp2_conn_set_local_transport_params_raw()` 函数

### 2.2 lib/ngtcp2_conn.c
- **目的**：实现原始传输参数设置
- **修改点**：
  - 添加 `conn_clear_local_transport_params_raw()` 清理函数
  - `ngtcp2_conn_del()` — 释放transport_params_raw.base
  - `ngtcp2_conn_set_local_transport_params_versioned()` — 调用conn_clear_local_transport_params_raw
  - 实现 `ngtcp2_conn_set_local_transport_params_raw()` — 验证、解码、拷贝原始参数
  - `ngtcp2_conn_encode_local_transport_params()` — 优先使用raw参数，否则正常编码

### 2.3 lib/ngtcp2_conn.h
- **目的**：在连接结构中存储raw参数
- **修改点**：添加 `transport_params_raw` 字段到local结构

### 2.4 crypto/boringssl/libngtcp2_crypto_boringssl.pc.in
- **目的**：移除对openssl的pkg-config依赖
- **为什么**：避免静态链接时pkg-config拉入系统OpenSSL

## 三、nghttp3 修改（nghttp3.patch）

### 3.1 lib/includes/nghttp3/nghttp3.h
- **目的**：添加HTTP/3 SETTINGS控制API
- **为什么**：浏览器的HTTP/3 SETTINGS帧的顺序和值需要精确匹配指纹
- **修改点**：
  - 将SETTINGS ID常量移到公开头文件
  - 添加 `nghttp3_settings_entry` 结构体（id+value）
  - 声明 `nghttp3_conn_submit_settings()` 函数

### 3.2 lib/nghttp3_conn.c
- **目的**：实现自定义SETTINGS提交
- **修改点**：
  - `nghttp3_conn_del()` — 释放settings_iv
  - `nghttp3_conn_bind_control_stream()` — 如果有自定义settings，使用自定义而非默认
  - 实现 `nghttp3_conn_submit_settings()` — 验证、拷贝settings数组

### 3.3 lib/nghttp3_conn.h
- **目的**：存储自定义settings
- **修改点**：添加 settings_iv/settings_niv/settings_submitted 字段

### 3.4 lib/nghttp3_frame.h
- **目的**：将内部定义移到公开头文件
- **修改点**：移除SETTINGS ID常量和nghttp3_settings_entry（已移到nghttp3.h）

### 3.5 lib/nghttp3_stream.c
- **目的**：支持自定义SETTINGS帧写入
- **修改点**：`nghttp3_stream_write_settings()` — 使用传入的frame参数而非硬编码settings

## 七、修改优先级总结

### 必须修改（影响核心功能）
1. **BoringSSL** — TLS指纹伪装的基础API（extension_order/cipher_order/key_shares_limit等）
2. **curl/lib/impersonate.c/h** — 35+浏览器预设数据
3. **curl/lib/easy.c** — curl_easy_impersonate()主入口
4. **curl/include/curl/curl.h** — 32个新CURLOPT定义
