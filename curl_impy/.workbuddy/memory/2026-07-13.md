# 2026-07-13 工作日志

## curl-impy SSL 验证 bug 修复

**问题**: `verify=True` 时 SSL 验证失败，用户需要 `verify=False` 才能正常请求。

**根因分析**:
1. `Curl.__init__` 未在初始化时设置 `CURLOPT_CAINFO`（curl-cffi 会设置）
2. `setopt()` 方法中，常规字符串选项处理分支在第 299 行 `return 0` 提前返回，导致末尾的 `if option == CurlOpt.CAINFO: self._is_cert_set = True` 永远不会执行 — `_is_cert_set` 标志始终为 False
3. `set_curl_options` (utils.py) 设置了 `c._skip_cacert` 属性，但 `perform()` 和 `_ensure_cacert()` 从未检查该属性 — 死代码

**修复内容** (curl.py):
1. `Curl.__init__`: 新增 `self.setopt(CurlOpt.CAINFO, self._cacert)` 调用，初始化时即设置 CA 证书路径
2. `setopt()` 字符串选项分支: 在 `return 0` 前添加 `if option == CurlOpt.CAINFO: self._is_cert_set = True`
3. `perform()` 和 `_ensure_cacert()`: 添加 `getattr(self, "_skip_cacert", False)` 检查，verify=False 时跳过 CAINFO 设置
4. `reset()`: 添加 `self._skip_cacert = False` 重置

**验证**: 同步路径 (Session) 全部通过 — verify=True/False 均正常工作，重复请求（reset 路径）也正常。异步路径在 Windows 上存在独立的 ProactorEventLoop 挂起问题，与本次修复无关。

## curl-impy 功能补全（对齐 curl-cffi）

### PROXY_CAINFO 补全
- `Curl.__init__`、`_ensure_cacert()`、`perform()` 中同时设置 `CAINFO` + `PROXY_CAINFO`
- `setopt()` 字符串分支：`_is_cert_set` 对 `CAINFO` 和 `PROXY_CAINFO` 均生效

### WebSocket 支持 (Curl 类)
- 新增 `CurlWsFrame` NamedTuple (const.py)，映射 libcurl `curl_ws_frame` 结构
- `Curl.ws_recv(buffer_size)` → `(bytes, CurlWsFrame)`
- `Curl.ws_send(payload, flags, framesize)` → `int`
- `Curl.ws_close()` → 发送 CLOSE 帧
- _ffi.py 已有 `curl_ws_recv`/`curl_ws_send` 声明，无需修改

### 工具方法
- `Curl.upkeep()` — 连接保活 (curl_easy_upkeep)
- `Curl.parse_cookie_headers(list[bytes])` → `SimpleCookie`
- `Curl.slist_to_list(slist_ptr)` — 静态方法，slist 转 list[bytes]
- `Curl.clean_handles_and_buffers()` — 提取自 perform()/cleanup() 的清理逻辑
- `bytes_to_hex(data, width)` — 模块级 hex dump 工具
- `debug_function_default(infotype, data)` — 默认 debug 回调

### CurlMime 改进
- `CurlMime.from_list(parts)` — 类方法，从 dict 列表创建
- `CurlMime.attach(curl)` — 设置 MIMEPOST 选项
- `CurlMime.close()` — free() 的别名
- `addpart()` 新增 `local_path` 参数（curl-cffi 兼容别名）
- `setopt()` 新增 `MIMEPOST` 特殊处理（传递 cdata 指针而非字符串）

### 版本元数据
- `__version__.py`: 新增 `__title__`, `__description__`，版本升至 0.4.0
- `curl.py`: 新增 `__curl_version__` (libcurl 版本字符串) 和 `is_pro()` 函数
- `__init__.py`: 导出所有新符号

### setopt MIMEPOST 修复
- MIMEPOST (10000+269) 落入字符串选项分支导致 `ffi.new("char[]", mime_ptr)` 报错
- 新增 MIMEPOST 特殊分支，直接传递 cdata 指针
