# curl-impersonate 升级构建文档

> 工作文档，随构建推进持续更新。

## 一、项目概述

**本项目 ≠ lexiforest/curl-impersonate！** 独立定制版，在 lexiforest 基础上扩展了自定义浏览器注册、ECH GREASE、ALPS 新 codepoint、N-API 绑定等功能。

### 三大核心 API

| API | 功能 | 状态 |
|-----|------|------|
| `curl_easy_impersonate_list()` | 列出可模拟的浏览器（内置 + 自定义） | 🔴 待实现 |
| `curl_easy_impersonate_register(target, json_config)` | 动态注册自定义浏览器指纹 | 🔴 待实现 |
| `curl_easy_impersonate(curl, target, default_headers)` | 使用指定浏览器指纹发起请求 | 🔴 待实现 |

### 验证网站

`https://120.26.33.71/json/detail`（⚠️ 自签名证书，需 `CURLOPT_SSL_VERIFYPEER=0` + `CURLOPT_SSL_VERIFYHOST=0`）

## 三、完整工作流程（9步法）

> **⚠️ 核心原则：所有源码全部要手动修改，即使旧的patch可以直接应用也不要用，必须全部手动修改，这样才能理解源码目的！**
>
> **⚠️ 旧构建脚本（win_build/*.bat/*.py）仅作为参考！参考其中的工具路径、工具使用方法、编译参数、链接参数。不要直接使用旧构建脚本构建！**
>
> **⚠️ 最终需要重写全新的构建脚本（Step 9）**

### Step 1: 清理项目目录 ✅

- 删除所有不需要参考的文件，清空整个目录
- **只保留**旧构建脚本（win_build/*.bat, *.py, *.def）用于后续参考
- 当前保留文件：
  - `Chrome144.json` / `browsers.json` — 浏览器指纹配置
  - `任务文档.md` / `修改文档.md` — 工作文档
  - `win_build/*.bat` — 构建脚本（仅参考工具路径和用法）
  - `win_build/*.py` — 构建辅助脚本（仅参考工具路径和用法）
  - `win_build/*.def` — 导出定义文件（仅参考）

### Step 2: 下载原版依赖，建立git基线 🔴

- 按照上方版本表下载所有依赖源码到 `deps/` 目录
- 解压到对应的 `deps/<name>-<version>/` 目录
- **每个依赖都在其目录内初始化git**：`git init && git add -A && git commit -m "baseline"`
- 下载 lexiforest/curl-impersonate 用于参考（克隆到 `lexiforest_ref/`）
- 项目根目录也初始化git

### Step 3: 分析lexiforest修改点，建立修改文档 🔴

- 分析 lexiforest/curl-impersonate 的所有修改点
- 理解修改的意图、目的
- **不记录具体修改内容**，只记录修改的文件、函数、目的、意图、为什么要这样改
- 建立/更新修改文档.md
- **然后删除 lexiforest/curl-impersonate**（忘掉他，忘掉细节）

### Step 4: 按修改文档手动修改8.20.0源码 🔴

**所有修改必须手动做！不用patch apply！**

分为4大块修改：

**A. BoringSSL修改**（按修改文档第一章，手动修改 deps/boringssl/ 下的源码）：
1. CMakeLists.txt — Windows构建适配+移除不需要的组件
2. crypto/cipher/e_tls.cc — 添加AES-256-CBC+SHA256/SHA384 AEAD
3. crypto/fipsmodule/ec/p256_test.cc + third_party/fiat/p256_64.h — MSVC兼容
4. crypto/rand/windows.cc — 兼容旧版Windows
5. include/openssl/aead.h — 导出新AEAD函数声明
6. include/openssl/nid.h — 添加FFDHE NID
7. include/openssl/ssl.h — 添加TLS指纹伪装API
8. include/openssl/target.h — LoongArch64支持
9. include/openssl/tls1.h — TLS扩展类型常量
10. ssl/extensions.cc — TLS扩展控制核心
11. ssl/handshake_client.cc — 密码套件顺序+DHE+签名算法
12. ssl/internal.h — 新常量和结构体字段
13. ssl/ssl_cipher.cc — DHE/CBC/SHA384密码套件
14. ssl/ssl_key_share.cc — FFDHE密钥交换组
15. ssl/ssl_lib.cc — 新API实现
16. ssl/ssl_privkey.cc — 移除签名算法唯一性检查

**B. ngtcp2修改**（按修改文档第二章，手动修改 deps/ngtcp2-1.20.0/）：
1. lib/includes/ngtcp2/ngtcp2.h — 原始传输参数API
2. lib/ngtcp2_conn.c — 实现原始传输参数设置
3. lib/ngtcp2_conn.h — 存储raw参数
4. crypto/boringssl/libngtcp2_crypto_boringssl.pc.in — 移除openssl依赖

**C. nghttp3修改**（按修改文档第三章，手动修改 deps/nghttp3-1.15.0/）：
1. lib/includes/nghttp3/nghttp3.h — HTTP/3 SETTINGS API
2. lib/nghttp3_conn.c — 自定义SETTINGS
3. lib/nghttp3_conn.h — 存储自定义settings
4. lib/nghttp3_frame.h — 移动常量到公开头
5. lib/nghttp3_stream.c — 自定义SETTINGS帧写入

**D. brotli修改**（按修改文档第四章，手动修改 deps/brotli-1.2.0/）：
1. c/common/platform.h — IA64/LoongArch64兼容

**E. curl修改**（按修改文档第五章，手动修改 deps/curl-8.20.0/）— **最核心最大**：

1. **新增文件**：创建 lib/impersonate.c（35+浏览器预设数据表）和 lib/impersonate.h（impersonate_opts结构体）
2. **include/curl/curl.h**：添加32个新CURLOPT定义（需确认编号不与8.20.0原生冲突）
3. **include/curl/easy.h**：声明 curl_easy_impersonate() / register() / list()
4. **include/curl/typecheck-gcc.h**：为新CURLOPT添加类型检查宏
5. **lib/urldata.h**：添加新选项对应的字段到 struct UserDefined
6. **lib/setopt.c**：添加新CURLOPT的处理逻辑
7. **lib/easy.c**：新增 curl_easy_impersonate() 函数实现
8. **lib/easyoptions.c**：添加新CURLOPT选项条目+更新计数检查
9. **lib/vtls/openssl.c**：根据impersonate选项调用BoringSSL自定义API
10. **lib/http2.c**：HTTP/2指纹控制
11. **lib/http.c**：HTTP头顺序/基础头/cookie分割
12. **lib/vquic/curl_ngtcp2.c**：QUIC指纹集成
13. **lib/vquic/vquic.c**：QUIC TLS选项
14. **lib/vtls/vtls.c/h**：传递impersonate参数
15. **lib/其他**：dynhds/mime/multi/socks/transfer/url/ws 等

**F. 项目自定义扩展**（非lexiforest）：
16. 创建 lib/cJSON.c/h — JSON解析器
17. 创建 lib/impersonate_register.c/h — 自定义浏览器注册
18. 修改 impersonate.h — 添加 ech_grease + alps_use_new_codepoint

### Step 5: 检查源码确保三大函数能正常运行 🔴

- `curl_easy_impersonate()` — 检查预设表完整、setopt调用链完整
- `curl_easy_impersonate_register()` — 检查JSON解析、注册表、查找逻辑
- `curl_easy_impersonate_list()` — 检查列表生成、内存分配
- 检查所有新CURLOPT是否在 setopt.c/easyoptions.c/urldata.h 三处一致
- 修复因修改带来的BUG

### Step 6: 用现有源码构建 🔴

> ⚠️ 参考旧构建脚本中的工具路径和方法，但不要直接运行旧脚本！
> 从deps/现有源码构建，不重新下载解压。

构建顺序：
1. zlib → `install/zlib/lib/zlibstatic.lib`
2. brotli → `install/brotli/lib/brotlidec-static.lib` 等
3. nghttp2 → `install/nghttp2/lib/nghttp2.lib`
4. ngtcp2 → `install/ngtcp2/lib/ngtcp2.lib` + `ngtcp2_crypto_boringssl.lib`
5. nghttp3 → `install/nghttp3/lib/nghttp3.lib`
6. zstd → `install/zstd/lib/zstd_static.lib`
7. BoringSSL → `install/boringssl/lib/libssl.lib` + `libcrypto.lib`
8. curl lib → `output/libcurl-impersonate.lib`
9. curl dll → `output/libcurl-impersonate.dll`

关键构建参数（从旧脚本参考）：
- CMake + Ninja 构建
- VS 2019/2022 工具链（find_vs.bat查找）
- `/MT` 静态链接C运行时（不依赖MSVC运行时DLL）
- 静态链接所有依赖库

## 四、旧构建脚本参考信息

> ⚠️ 以下仅作为参考，了解工具路径和使用方法。不要直接使用这些脚本。

### 前置条件
Windows 10/11 x64 / VS 2019/2022 / CMake ≥ 3.20 / Ninja / Git / NASM / Python 3.x / Node.js

### 关键工具路径
- `find_vs.bat` — 查找VS安装路径（替代vswhere）
- `detect_env.bat` — 检测构建环境，设置VSBINDIR/VCINSTALLDIR等变量
- `config.bat` — 设置构建目录/安装目录/架构等配置

### 关键编译参数
- CMake Generator: `Ninja`
- C运行时: `/MT`（静态链接，不依赖MSVC运行时DLL）
- BoringSSL C++运行时: 静态链接
- 输出库名: `libcurl-impersonate`（不是libcurl）

### 构建依赖顺序
zlib → brotli → nghttp2 → ngtcp2 → nghttp3 → zstd → BoringSSL → curl
